## 😵 블록 기준점이 구석에 있어서 배치할 때마다 짜증나요

블록 작업하다 보면 이런 황당한 상황 많이 겪으시죠?

**상황:** 의자 블록을 배치하려고 클릭했는데...

```
클릭한 위치: 의자 중심 (내 기대)
실제 배치 위치: 의자 다리 끝!!! 😱
```

**왜?**

→ 블록 기준점(Basepoint)이 **구석에 있어서!**

**문제점:**
- 배치할 때마다 위치 조정 필요
- 회전 시키면 영뚱한 곳을 중심으로 돔
- 스케일 조정하면 더 이상해짐
- 정렬(ALIGN) 명령도 제대로 안 됨

**해결법은?**

1. 블록 편집기 들어가서
2. 기준점 수동으로 중심으로 이동
3. 기존 배치된 블록들 위치 전부 수정...

**아니, 이게 무슨 고문이야!** 💢

그래서 만들었습니다.

> "클릭 한 번에 블록 기준점을 중심으로 옮길 수 없을까?"

그 답이 **BCENT 리습**입니다! 🎯

---

## ⚡ BCENT 리습이 하는 마법

**기존 방법:**
```
1. 블록 편집기 (BEDIT) 열기
2. 기준점 수동 계산
3. 객체들 이동
4. 블록 재정의
5. 기존 배치된 블록들 위치 수정...

소요 시간: 블록 1개당 5분 😭
```

**BCENT 방법:**
```
1. 명령어 BCENT 입력
2. 블록 선택
3. 자동 처리!

소요 시간: 5초 ⚡
```

**무려 60배 빠름!** 🚀

---

## 📖 사용 방법 (초간단!)

### 1단계: LISP 로드

`APPLOAD` → `AN_BCENT_블록중심기준점 자동지정.lsp` 로드

### 2단계: 명령어 실행 및 선택

```
명령: BCENT

=== 블록 기준점 중심으로 변경 ===
여러 종류의 블록을 선택할 수 있습니다.
[블록들 선택!]
```

### 3단계: 자동 처리 확인

```
선택된 블록 개수: 15

블록 'CHAIR-01' 처리 중...
    현재 기준점: (0.00, 0.00)
    새 기준점: (250.00, 200.00)
  ✓ 'CHAIR-01' 기준점이 중심으로 변경되었습니다.

블록 'TABLE-01' 처리 중...
    현재 기준점: (50.00, 0.00)
    새 기준점: (400.00, 300.00)
  ✓ 'TABLE-01' 기준점이 중심으로 변경되었습니다.

=== 완료 ===
총 2개의 서로 다른 블록이 처리되었습니다.
처리된 블록:
  - CHAIR-01
  - TABLE-01
```

**끝!** 블록 기준점이 중심으로 이동! 🎉

## 🎯 실전 활용 예시

### 예시 1: 가구 블록 배치

**Before (기준점이 구석):**
```
의자 클릭 → 다리 끝에 배치됨
테이블 클릭 → 모서리에 배치됨

**BCENT 실행 후:**
```
의자 클릭 → 중심에 배치됨! ✅
테이블 클릭 → 중심에 배치됨! ✅
```

**배치가 완전 직관적으로!** 👍

```

```

```

## 💡 똑똑한 기능들

### 기능 1: 바운딩 박스로 자동 계산

```
블록의 모든 객체를 감싸는 박스의 중심점 자동 계산!
수동 계산 필요 없음!
```

**정확도 100%!** 🎯

### 기능 2: 회전/스케일 완벽 보정

```
기존에 배치된 블록들:
- 회전 각도
- X/Y 스케일
- 삽입점

모두 고려해서 위치 자동 보정!
```

**배치된 블록 위치 그대로 유지!** ✅

### 기능 3: 여러 종류 동시 처리

```
의자 10개, 테이블 5개 선택
→ 의자, 테이블 각 1번씩만 처리!

중복 처리 안 함 = 빠름!
```

**똑똑하죠?** 🧠

### 기능 4: 처리 과정 실시간 표시

```
현재 기준점: (0.00, 0.00)
새 기준점: (250.00, 200.00)
```

**뭐가 바뀌는지 한눈에!** 👀

## 🔥 이 리습의 장점

✅ **자동 계산** - 중심점 수동 계산 불필요
✅ **위치 보정** - 배치된 블록 위치 유지
✅ **회전/스케일 고려** - 완벽한 보정
✅ **중복 제거** - 같은 블록은 1번만 처리
✅ **실시간 로그** - 처리 과정 표시

## 📊 작업 프로세스

**BCENT가 하는 일 (자동):**

```
1. 블록 이름 확인
2. 임시로 (0,0,0)에 블록 삽입
3. 바운딩 박스로 중심점 계산
4. 블록 분해 (EXPLODE)
5. 분해된 객체들을 중심 기준으로 이동
6. 새 블록으로 재정의
7. 도면의 모든 해당 블록 인스턴스 업데이트
   - 회전 각도 고려
   - 스케일 고려
   - 새 삽입점 계산
8. REGEN으로 화면 업데이트
```

**8단계를 5초만에!** ⚡
