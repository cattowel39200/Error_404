## 🤦‍♂️ 이 리습을 만들게 된 계기

AutoCAD로 작업하다 보면 블록 이름을 바꿔야 할 때가 생각보다 자주 있어요.

"아, 이 블록 이름을 'CHAIR01'에서 'CHAIR_TYPE-A'로 바꿔야 하는데..."

보통은 어떻게 하나요?

1. **블록 편집기 들어가서** 블록 재정의하거나
2. **RENAME 명령어** 쓰거나
3. 심지어 **블록 삭제하고 다시 만들거나** 😱

특히 블록이 도면에 100개, 200개 깔려있을 때... 블록 재정의하면 모든 인스턴스가 영향받을까 봐 조마조마하죠.

저도 처음에는 RENAME 명령어를 썼는데, 문제는 **타이핑이 너무 귀찮다**는 거예요.

```
명령: RENAME
이름을 바꿀 객체 종류 입력: BLOCK
기존 블록 이름: CHAIR01
새 블록 이름: CHAIR_TYPE-A
```

이거 매번 하면 손목이 아프더라고요. 😤

그래서 만들었습니다. **클릭 한 번, 타이핑 한 번이면 끝나는 BRN 리습!**

---

## ⚡ BRN 리습이 뭐하는 앤가요?

간단합니다.

1. **블록을 클릭**하면
2. **현재 이름이 표시**되고
3. **새 이름 입력**하면 끝!

심지어 기존 이름이 `<CHAIR01>` 이렇게 기본값으로 나와서, **수정만 하면 돼요**. 처음부터 다시 칠 필요 없음!

---

## 📖 사용 방법 (초간단 3단계)

### 1단계: LISP 로드

AutoCAD에서 `APPLOAD` 명령어 입력 → `AN_BRN_블록명변경.lsp` 파일 선택 → 로드!

### 2단계: 명령어 실행

```
명령: BRN
```

### 3단계: 블록 선택하고 이름 입력

```
이름을 변경할 블록을 선택하세요: [블록 클릭!]
현재 블록 이름: CHAIR01
새 블록 이름 입력 <CHAIR01>: CHAIR_TYPE-A ↵
블록 이름이 'CHAIR01'에서 'CHAIR_TYPE-A'으로 변경되었습니다.
```

**끝!**

도면에 있는 모든 'CHAIR01' 블록이 자동으로 'CHAIR_TYPE-A'로 바뀝니다. 👏

---

## 🎯 실전 활용 예시

### 상황 1: 블록 이름 규칙 변경

회사 도면 표준이 바뀌어서 블록 이름 규칙을 변경해야 할 때:

- `TB01` → `TABLE-01`
- `CH01` → `CHAIR-01`
- `DK01` → `DESK-01`

**BRN 리습으로 하나씩 클릭만 하면 5초 만에 완료!** ⚡

### 상황 2: 오타 수정

```
현재: CHIAR01 (오타 발견!)
수정: CHAIR01
```

BRN 명령어 → 클릭 → 수정 → 완료!

### 상황 3: 이름만 확인하고 싶을 때

```
새 블록 이름 입력 <CHAIR01>: ↵ (그냥 Enter)
기존 블록 이름을 유지합니다.
```

**Enter만 누르면 변경 없이 종료**되니까, 블록 이름 확인용으로도 쓸 수 있어요!

## 💡 꿀팁 모음

### 팁 1: 중복 이름 자동 감지

이미 존재하는 블록 이름을 입력하면?

```
새 블록 이름 입력 <CHAIR01>: TABLE01
이미 존재하는 블록 이름입니다. 다시 입력하세요.
```

**에러로 도면 날려먹을 걱정 없음!** 👍

### 팁 2: 띄어쓰기도 OK

```
새 블록 이름 입력 <CHAIR01>: CHAIR TYPE A
```

띄어쓰기 포함된 이름도 가능합니다. (단, AutoCAD에서는 되도록 `-`나 `_` 사용 권장)

### 팁 3: 기존 블록 정의는 자동 삭제

변경 후 기존 블록 정의(`CHAIR01`)는 **PURGE 명령어로 자동 삭제** 시도합니다.

도면 파일 용량도 깔끔하게 관리! 🧹
