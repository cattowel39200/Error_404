## 🥇 1위: 에러 메시지를 읽어라 (정말로!)

### 과거의 나

```bash
Error: Cannot find module 'express'
    at Function.Module._resolveFilename (internal/modules/cjs/loader.js:xxx)
    at Function.Module._load (internal/modules/cjs/loader.js:xxx)
    ...
```

**반응:** "에러다! 뭔가 잘못됐어! 어떡하지!"
*(에러 메시지는 안 읽고 패닉)*

### 지금의 나

같은 에러를 보고:

**1. 첫 줄을 읽는다**
```
Error: Cannot find module 'express'
```

**2. 해석한다**
- "모듈을 찾을 수 없다"
- 그 모듈은 'express'

**3. 해결한다**
```bash
npm install express
```

끝!

### 💡 핵심 교훈

```
에러 메시지 = 컴퓨터가 준 힌트
```

대부분의 에러 메시지는:
- ✅ **첫 줄**: 무엇이 문제인지
- ✅ **두 번째 줄**: 어디서 문제가 생겼는지
- ✅ **나머지**: 디버깅용 스택 트레이스

**초보자 팁:**
1. 에러 메시지 전체를 **천천히** 읽기
2. 첫 줄에 집중하기
3. 모르는 단어는 구글 번역
4. 그대로 구글에 검색 (한글 번역 X)

## 🥈 2위: 구글링은 기술이다

### 과거의 나

**검색어:** "안됨"

**검색어:** "파이썬 에러"

**검색어:** "코드 이상함"

→ 당연히 원하는 답을 못 찾음

### 지금의 나

**좋은 검색어 작성법:**

#### ❌ 나쁜 예
```
"React 안 돼요"
"npm 에러"
"왜 안 되지?"
```

#### ✅ 좋은 예
```
"react useEffect dependency warning"
"npm ERR EACCES permission denied"
"javascript array map is not a function"
```

### 💡 핵심 교훈

**구글링 공식:**

```
[기술명] + [에러 키워드] + [영어]
```

**예시:**
- Python + "ModuleNotFoundError" + pandas
- React + "hooks" + "can't perform a React state update"
- Git + "fatal: not a git repository"

**추가 팁:**

1. **에러 메시지를 그대로 복사** (경로는 제거)
   ```
   ❌ Error in C:\Users\MyName\project\index.js:15
   ✅ Error: Cannot read property 'map' of undefined
   ```

2. **Stack Overflow 필터**
   ```
   검색어 + site:stackoverflow.com
   ```

3. **최신 정보 찾기**
   ```
   검색어 + 2024 (또는 2025)
   ```

4. **공식 문서 우선**
   ```
   검색어 + docs (또는 documentation)
   ```

## 🥉 3위: 공식 문서를 친구로 만들어라

### 과거의 나

"공식 문서? 어렵고 영어로 되어있잖아. 블로그 튜토리얼이나 볼래."

→ 블로그 따라했는데 버전이 달라서 안 됨
→ 다른 블로그 찾기
→ 또 안 됨
→ 무한 반복

### 지금의 나

**공식 문서 읽는 법:**

1. **Getting Started부터** 시작
   - 대부분 초보자용 가이드가 있음
   - "Quick Start", "Tutorial" 섹션 찾기

2. **예제 코드 복사해서 실행**
   - 이론보다 실습이 먼저
   - 일단 돌아가는 걸 보고 이해하기

3. **모르는 단어는 체크**
   - 한꺼번에 이해하려 하지 말기
   - 중요한 개념만 메모

### 추천 공식 문서

**입문자 친화적인 문서들:**

- [MDN Web Docs](https://developer.mozilla.org) - JavaScript, HTML, CSS
  - 한국어 지원 ✅
  - 예제 코드 풍부 ✅
  - 초보자 친화적 ✅

- [React 공식 문서](https://react.dev)
  - 새 문서는 정말 쉬움
  - 인터랙티브 예제

- [Python 공식 튜토리얼](https://docs.python.org/ko/3/tutorial/)
  - 한국어 번역 있음
  - 체계적인 구성

### 💡 핵심 교훈

```
블로그 튜토리얼: 빠른 시작
공식 문서: 정확한 이해
```

**조합 전략:**
1. 블로그로 빠르게 시작
2. 막히면 공식 문서 확인
3. 더 깊이 알고 싶으면 공식 문서 정독

## 🏅 4위: 작은 단위로 테스트하라

### 과거의 나

100줄 코드를 한 번에 작성
→ 실행
→ 에러 10개
→ 어디가 문제인지 모름
→ 멘붕

### 지금의 나

**코드 작성 → 즉시 테스트 → 다음 코드**

```javascript
// Step 1: 데이터 준비
const users = [
  { name: "Alice", age: 25 },
  { name: "Bob", age: 30 }
];
console.log(users); // ✅ 확인

// Step 2: 맵핑
const names = users.map(user => user.name);
console.log(names); // ✅ 확인

// Step 3: 필터링
const adults = users.filter(user => user.age >= 20);
console.log(adults); // ✅ 확인

// 이렇게 단계별로!
```

### 💡 핵심 교훈

**작은 성공의 반복 > 한 번에 큰 성공**

**실천 방법:**

1. **console.log() 남발**
   ```javascript
   console.log("여기까지 왔나?");
   console.log("data:", data);
   console.log("result:", result);
   ```

2. **주석으로 계획 세우기**
   ```javascript
   // 1. 데이터 가져오기

   // 2. 데이터 가공하기

   // 3. 화면에 표시하기

   // 한 번에 하나씩 구현!
   ```

3. **Git으로 자주 커밋**
   ```bash
   git add .
   git commit -m "Step 1: 데이터 가져오기 완료"
   # 뭔가 잘못되면 되돌리기 쉬움
   ```

## 🎖️ 5위: 완벽한 코드는 없다 (일단 돌아가게 만들어라)

### 과거의 나

"이 코드는 완벽하지 않아. 더 좋은 방법이 있을 거야."
→ 2시간 동안 리팩토링 고민
→ 결국 아무것도 안 함

### 지금의 나

```javascript
// V1: 일단 돌아가게 (10분)
function getSum(arr) {
  let sum = 0;
  for (let i = 0; i < arr.length; i++) {
    sum += arr[i];
  }
  return sum;
}

// 작동 확인 ✅

// V2: 나중에 개선 (여유 있을 때)
function getSum(arr) {
  return arr.reduce((sum, num) => sum + num, 0);
}
```

### 💡 핵심 교훈

**"Make it work, make it right, make it fast"**
- 1단계: 일단 작동하게 만들기
- 2단계: 올바르게 만들기
- 3단계: 빠르게 만들기

**초보자는 1단계에 집중!**

**실천 방법:**

1. **일단 작동하는 코드 작성**
   - 복잡하게 생각하지 말기
   - for문이 편하면 for문 사용

2. **작동 확인 후 개선**
   - 돌아간다는 확신이 있을 때 리팩토링
   - Git으로 이전 버전 보존

3. **"나중에" 목록 만들기**
   ```javascript
   // TODO: 나중에 reduce로 바꾸기
   // TODO: 에러 처리 추가
   // TODO: 성능 개선
   ```

## 🎁 보너스: 휴식도 실력이다

### 과거의 나

에러가 안 풀려서 5시간 동안 버티기
→ 눈은 충혈, 머리는 지끈
→ 결국 못 풀음

### 지금의 나

30분 해결 안 되면:
1. 커피 한 잔 ☕
2. 산책 10분 🚶
3. 완전히 다른 일 하기

→ 돌아와서 5분 만에 해결

### 💡 핵심 교훈

**뇌도 쉬어야 작동한다**

**추천 루틴:**

```
25분 코딩 → 5분 휴식 (뽀모도로 기법)
```

**막혔을 때:**
1. 일단 저장하고 닫기
2. 완전히 다른 것 하기
3. 다음 날 다시 보기

놀랍게도 **자는 동안 뇌가 알아서 해결**하는 경우가 많습니다.
