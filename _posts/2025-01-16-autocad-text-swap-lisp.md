## 😩 복사 붙여넣기 지옥에서 탈출하고 싶었어요

도면 작업하다 보면 **두 텍스트의 내용을 서로 바꿔야** 할 때가 있죠?

예를 들어:
- 테이블에서 두 칸의 숫자를 서로 바꾸기
- 치수 주석 위치 변경하기
- 실수로 잘못 입력한 텍스트 순서 바로잡기

저는 이럴 때마다 이렇게 했어요:

```
1. 첫 번째 텍스트 → 더블클릭 → 전체 선택 → Ctrl+C
2. 메모장 열기 → 붙여넣기 (임시 저장)
3. 두 번째 텍스트 → 더블클릭 → 전체 선택 → Ctrl+C
4. 첫 번째 텍스트로 돌아가기 → 더블클릭 → Ctrl+V
5. 메모장 내용 복사 → 두 번째 텍스트 → 더블클릭 → Ctrl+V
```

**5단계!** 😱

이게 한두 개면 괜찮은데, 테이블에서 10개, 20개 바꿔야 하면 **진짜 손목 터져요**.

그래서 고민했습니다.

> "클릭 두 번만 하면 내용이 바뀌면 안 될까?"

그렇게 탄생한 게 **TTC 리습**입니다! 🎉

---

## ⚡ TTC 리습이 뭐하는 애인가요?

엄청 간단해요.

1. **첫 번째 텍스트 클릭**
2. **두 번째 텍스트 클릭**
3. **끝!**

두 텍스트의 내용이 **자동으로 서로 바뀝니다**. 마법처럼! ✨

TEXT든 MTEXT든 상관없이 다 됩니다!

---

## 📖 사용 방법 (클릭 2번이면 끝)

### 1단계: LISP 로드

AutoCAD에서 `APPLOAD` → `AN_TTC_글자교환.lsp` 로드

### 2단계: 명령어 실행

```
명령: TTC
```

또는

```
명령: TTS
```

(둘 다 같은 기능이에요!)

### 3단계: 텍스트 두 개 클릭

```
첫번째 텍스트 객체를 선택하세요: [텍스트1 클릭]
두번째 텍스트 객체를 선택하세요: [텍스트2 클릭]
두 텍스트 객체의 내용이 교환되었습니다.
```

**2초 컷!** ⚡

---

## 🎯 실전 활용 예시

### 예시 1: 테이블 데이터 순서 변경

**시공 전:**

| 구역 | 수량 |
|------|------|
| A구역 | 150 |
| B구역 | 200 |

근데 실제로는 A가 200, B가 150이어야 함! 😨

**TTC 리습 사용:**
```
명령: TTC
[150 클릭] → [200 클릭] → 완료!
```

**시공 후:**

| 구역 | 수량 |
|------|------|
| A구역 | 200 |
| B구역 | 150 |

**걸린 시간: 2초** 👍

---

### 예시 2: 치수 주석 위치 교정

평면도에서 두 치수의 주석 내용이 바뀌어야 할 때:

```
잘못됨:
왼쪽 벽체: "3,500"
오른쪽 벽체: "4,200"

TTC로 교환:
명령: TTC → 클릭 → 클릭

올바름:
왼쪽 벽체: "4,200"
오른쪽 벽체: "3,500"
```

**메모장 필요 없음!** 🎉

---

### 예시 3: 범례 항목 순서 바꾸기

도면 범례에서 항목 순서를 변경할 때도 유용해요:

```
기존:
1. 콘크리트
2. 아스팔트

변경 후:
1. 아스팔트
2. 콘크리트
```

TTC 명령어 → 클릭 두 번 → **완벽!** ✨

## 💡 꿀팁 & 주의사항

### 팁 1: TEXT와 MTEXT 섞어도 OK

```
첫 번째: TEXT 객체 (단일행 텍스트)
두 번째: MTEXT 객체 (여러행 텍스트)
```

**상관없이 교환 가능!** 👏

### 팁 2: 명령어 2개 (TTC, TTS)

`TTC`든 `TTS`든 아무거나 편한 거 쓰세요.

저는 개인적으로 `TTC` (Text To Change) 많이 써요!

### 팁 3: 텍스트 아닌 거 선택하면?

```
첫번째로 선택한 객체가 텍스트 객체가 아닙니다.
```

친절하게 에러 메시지 나와요. 안전함! 🛡️

### ⚠️ 주의: 한글이 깨지는 경우

만약 한글이 깨진다면 **인코딩 문제**일 수 있어요.

LISP 파일을 메모장으로 열고 **UTF-8로 다시 저장**하면 해결됩니다!

## 🔥 이 리습의 장점

✅ **초고속** - 클릭 2번, 2초 완료
✅ **간편함** - 메모장 필요 없음
✅ **안전함** - 텍스트 아니면 에러 표시
✅ **호환성** - TEXT, MTEXT 모두 지원
✅ **직관적** - 설명 안 봐도 사용 가능
